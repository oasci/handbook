# Project template

!!! info

    Here is a template that OASCI uses to plan and guide code-based projects.
    Thoughtful completion of this proposal is also the first step to proposing a successful project collaboration with OASCI.
    Remove this admonition and all annotations when drafting the proposal.

<div class="annotate" markdown>
-   **Title:** `<code name>`: `<one-sentence project description>` (1)
-   **Type:** Software
-   **[DRI](/organization/roles/leadership/dri):** `<OASCI member's name>` (2)
    { .annotate }

</div>

1.  We define the project's `<code name>` that will eventually become the software's name.
    This should be a unique name that is available in both the OASCI project space and is relatively absent in search engines.
    You should also check that the name is available on major platforms specific to the project's community.

    !!! example
        Check [PyPI](https://pypi.org/) and [conda](https://anaconda.org/) for Python projects.

    The `one-sentence project description` is self explanatory.
    It should be broad enough to be search-engine optimized, but specific enough to communicate the purpose of the project without reading the entire `README` file.

2.  The name of an OASCI [Fellow](/organization/scope/4-fellow) or [Director](/organization/scope/5-director) that is well-suited and agreed to be the Directly Responsible Individual (DRI) for the project.
